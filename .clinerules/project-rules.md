# Astra 项目级规则

## 1. 双语文档同步检查
当项目同时维护 EN 和 ZH 版 README 时，每次内容变更后必须逐段对照两份文件，确保：
- 每个顶级章节在另一语言版本中有对应的 1:1 翻译
- 章节编号在两份文件中保持一致
- 项目结构树、测试数量等硬数据在两份文件中同步更新

## 2. 以 ROADMAP.md 为权威参考
README 和文档中的 Phase 划分、功能列表、项目结构树，一律以 ROADMAP.md 为权威来源。任何对 README 结构树的变更，必须先对照 ROADMAP.md 确认条目完整。

## 3. 变更完成后的状态汇报
每次变更操作结束后，必须明确汇报以下三类状态：
### 离线（本地）
列出所有已修改/新增但尚未 commit 和 push 的文件（`git status --short`）。
### 在线（已推送）
列出已 commit 并 push 到远程的分支/commit。
### PR 状态
- **所有涉及文件修改的任务完成后，都必须创建 PR 并给出链接**（任何变更必须通过 PR 合并到 main，禁止直接推送到 main）。
- 如果 PR 已存在，给出 PR 链接及当前 CI 状态。
